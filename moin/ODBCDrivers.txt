= ODBC Drivers =
This page provides resources for people looking for ODBC drivers which they can then use with one of the available [[ODBC|Python ODBC interfaces]].

We group drivers by database backend. Where available, please add the database vendor drivers as first entry in each section.

<<TableOfContents()>>

== Generic Lists ==
 * [[https://web.archive.org/web/20140209203657/http://www.sqlsummit.com/ODBCVend.HTM|SQL Summit list of ODBC drivers and vendors]] <<BR>> This was once the most comprehensive listing of ODBC drivers. Unfortunately, the site stopped working in 2014, so the above is a link to the last archive.org version. Even though it is not updated anymore, it still provides a good overview of where to look for drivers.

 * [[http://www.databasedrivers.com/odbc/|ODBC Drivers on databasedrivers.com]]
 Listing of available ODBC drivers. This is also available as [[http://www.databasedrivers.com/sources/|list based on database backend]].

== ODBC Driver Vendors ==
There are a couple of companies which focus on creating commercial quality ODBC drivers for various backends:

 * [[http://www.actualtech.com/|Actual Technologies]] (specialized on Mac OS X)
 
 * [[http://www.cdata.com/|CData Software]] ODBC Drivers for more than 60 data sources.

 * [[https://www.devart.com/odbc/|Devart]] (Windows, Linux, Mac OS X)

 * [[http://www.datadirect.com/|DataDirect]]
 
 * [[http://www.easysoft.com/|EasySoft]]

 * [[http://www.openlinksw.com/|OpenLink]]

 * [[http://www.simba.com/connectors|Simba Technologies]] Drivers and driver development tools for Windows, Linux, Mac OS X: SQL Server, Couchbase, MongoDB, Hive, DynamoDB, Redshift, Cassandra, Spark, Impala, BigQuery, HBase, Salesforce.
